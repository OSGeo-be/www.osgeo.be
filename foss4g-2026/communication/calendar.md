## Automation options

### Option 1 έΑΦ Manual with structure (recommended to start)
- This file as a shared checklist on GitHub
- Comms lead checks off and publishes
- GitHub Issues with due dates for each send

### Option 2 έΑΦ Semi-automated with GitHub Actions
- A `cron` workflow auto-creates a GitHub Issue on each key date
- The Issue contains the text to publish
- The comms lead publishes manually έΗΤ checks off έΗΤ closes the Issue
- See: [automation/github-actions.md](../automation/github-actions.md)

### Option 3 έΑΦ Social automation (Mastodon + Matrix)
- GitHub Action έΗΤ Mastodon API (fosstodon.org) for short posts
- GitHub Action έΗΤ Matrix webhook for channel reminders
- Facebook/LinkedIn require an approved app (more complex)
